# Scientific Code Quality

Use these rules when implementing research-related functionality.

## Reproducibility

- Make jobs repeatable.
- Store input parameters for digest generation.
- Store timestamps and source URLs.
- Avoid relying on mutable global state.
- Prefer deterministic fallbacks when LLMs are unavailable.

## Inspectability

For every recommendation, store enough information to answer:

- Why was this selected?
- Which profile topics matched?
- Which negative topics penalized it?
- Was it boosted by author/lab watchlist?
- Was it boosted by prior feedback?
- Was the explanation generated by an LLM or fallback logic?

## Evaluation

When adding ranking changes, include at least one simple test showing that:

- relevant profile keywords increase scores
- negative topics decrease scores
- recent papers score higher than stale papers
- feedback can affect future ranking

## Avoid Overfitting

Do not make the system too dependent on a single keyword.
Use a combination of title, abstract, authors, categories, profile topics, Notion context, and feedback.
